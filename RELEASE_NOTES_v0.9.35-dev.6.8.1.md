# TaskFlow v0.9.35-dev.6.8.1 — Release Notes

**Дата:** 08.07.2026
**Тип:** UX bugfix (flash-of-free) + возврат потерянной вкладки Синхронизация
**Ветка:** develop

---

## TL;DR

Устранено визуальное мелькание («flash-of-free»): при переходе на вкладки **Подписка** и **Календарь** на ~1 секунду показывалось состояние для free-пользователя (CTA «Попробуйте Pro бесплатно» и paywall-заглушка «Раздел на Pro»), после чего экран переключался на актуальный. Теперь до подтверждения статуса подписки показывается нейтральное состояние без ложного контента.

Дополнительно: возвращена вкладка **Синхронизация** в боковом меню Настроек — она была случайно потеряна при перестановке порядка вкладок (компонент и роутинг работали, но пункта в меню не было).

---

## Проблема

При навигации на вкладку компонент монтируется заново, и хук `useEntitlement` снова стартует с `loading=true`. Оптимистичный старт берёт данные из локального SQLite-кэша (`entitlement_cache_v1`). Если кэш пуст или ещё не подтверждён (например сразу после логина или на новом устройстве), `effectivePlan` временно равен `free`, и UI успевал отрисовать финальное free-состояние **до** завершения fetch/realtime.

Два проявления одного класса бага:

1. **Вкладка Подписка** — блок Trial CTA «Попробуйте Pro бесплатно» рендерился по условию `{canStartTrial && (...)}` без guard'а загрузки. Бейдж плана рядом уже был защищён `subsLoading`, а CTA — нет.
2. **Вкладка Календарь** — обёрнута в `<PaywallGate>`. Гейт использовал `auth.loading` как единственный guard, но игнорировал `loading` из `useEntitlement`. При пустом кэше показывал paywall «Раздел на Pro», пока fetch не подтвердит Pro.

## Что изменено

### 1. `src/components/PaywallModal.tsx` — `PaywallGate`

- Добавлен guard: пока `loading === true` **и** вычисленный доступ был бы «denied» (free/trial-expired), рендерится **нейтральный лоадер** (маленький спиннер), а не paywall-заглушка.
- Paywall «Раздел на Pro» показывается **только когда `loading === false`** — тогда вердикт free окончательный.
- Порядок гейтов: `auth.loading` → children (splash в App.tsx); `e2e=1` bypass; `allowed` → children; `loading` → спиннер; иначе → paywall.
- Удалён теперь-недостижимый блок `{loading && !entitlement && …}` внутри paywall.
- Обновлён устаревший комментарий, который признавал мелькание как «ok для dev.6».

### 2. `src/pages/Settings.tsx` — Trial CTA (SubscriptionSection)

- Условие рендера `{canStartTrial && (...)}` → `{!subsLoading && canStartTrial && (...)}`.
- `subsLoading = auth.loading || entLoading` — тот же guard, что уже используется для остальных free-блоков секции (payment methods, оформление подписки и т.д.). Консистентно.

### 3. `src/pages/Settings.tsx` — вкладка Синхронизация

- В массив `navItems` возвращён пункт `{ key: 'sync' }` между `storage` (Хранилище) и `updates` (Обновления).
- Компонент `SyncSection` и рендер `{sub === 'sync' && <SyncSection />}` уже существовали — отсутствовал только пункт бокового меню. Прямой переход по коду работал и раньше.
- Label: «Синхронизация» / «Sync» (inline, в стиле соседних пунктов).

### 4. Версии

- `package.json`: `0.9.35-dev.6.8.0` → `0.9.35-dev.6.8.1`.
- `src-tauri/tauri.conf.json`: `0.9.35-dev.6.7.0` → `0.9.35-dev.6.8.1` (был рассинхронизирован, приведён к актуальной).
